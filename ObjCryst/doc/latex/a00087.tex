\subsubsection{Detailed Description}
class of refinable parameter types. 

This is used to categorize all parameters, to be able to modify a complete category of parameters (fix/unfix, set limits,...). The types are organized as a tree.

Parameters should be allocated globally in the heap, so we only use pointers in the interface.

\begin{DoxyNote}{Note}
when modifying (fixing, changing limits) for a given \doxyref{Ref\-Par\-Type}{p.}{a00087} (ie a 'family' of parameters), it affects all \doxyref{Refinable\-Par}{p.}{a00080} of this type {\itshape and} the parameters belonging to the children of this \doxyref{Ref\-Par\-Type}{p.}{a00087}. eg fixing for the type \char`\"{}gp\-Ref\-Par\-Type\-Scatt\char`\"{} will fix all the derived postionnal, orientationnal, and population parameters for the scatterers.
\end{DoxyNote}
\begin{DoxyRemark}{Remarks}
In the future, why not use a tree with multiple inheritance ?? It would be easy to allow multiple parents... But beware of loops... 
\end{DoxyRemark}
